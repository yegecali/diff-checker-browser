Usa la extensión para comparar los pasos de despliegue entre QA y Producción

Cómo usar: Abre la extensión → click en "Pick element" del Slot 1 → selecciona el recuadro QA → luego "Pick element" del Slot 2 → selecciona el recuadro PRODUCCIÓN. También puedes usar los selectores #env-qa y #env-prod.
Quality Assurance QA
1
Ejecutar Pipeline de CI/CD
Ingresar al pipeline de integración continua en Azure DevOps y disparar el build de la rama develop.
Organización : contoso-dev Proyecto : MiApp-QA Pipeline : deploy-qa Rama : develop Acción : Run pipeline → confirmar con "Run"
2
Validar ejecución del pipeline
Esperar que todas las etapas del pipeline finalicen con estado Succeeded. Revisar logs si alguna etapa falla.
✓ Stage: Build → Succeeded ✓ Stage: Unit Tests → Succeeded ✓ Stage: Docker Build → Succeeded ✓ Stage: Push to ACR → Succeeded ✓ Stage: Deploy to QA → Succeeded
3
Ingresar al Portal de Azure — Recurso QA
Navegar al portal de Azure y ubicar el grupo de recursos correspondiente al ambiente de QA.
Suscripción : Azure Dev/Test Resource Group : rg-miapp-qa-eastus AKS Cluster : aks-miapp-qa-001 Región : East US URL Portal : portal.azure.com → Kubernetes services
4
Ingresar al AKS — Validar Pods desplegados
Conectarse al clúster AKS de QA mediante kubectl y validar que los pods del namespace qa estén en estado Running.
# Conectar al cluster az aks get-credentials \ --resource-group rg-miapp-qa-eastus \ --name aks-miapp-qa-001 # Validar pods en namespace qa kubectl get pods -n qa -o wide # Resultado esperado NAME READY STATUS RESTARTS miapp-api-7d4f9b-xk2lp 1/1 Running 0 miapp-worker-6c8d2a-p9mnq 1/1 Running 0 miapp-scheduler-3b1e7f-r4wzj 1/1 Running 0
5
Ingresar a la Base de Datos — QA
Conectarse al servidor de base de datos de QA y ejecutar una consulta de validación para confirmar que el esquema y los datos de prueba están correctos.
Servidor : sql-miapp-qa.database.windows.net Base datos: db-miapp-qa Usuario : sqladmin-qa Puerto : 1433 -- Query de validación SELECT TOP 10 d.DeploymentId, d.Version, d.Environment, d.DeployedAt, d.Status FROM dbo.Deployments d WHERE d.Environment = 'QA' AND d.DeployedAt >= GETDATE() - 1 ORDER BY d.DeployedAt DESC; -- Resultado esperado DeploymentId Version Environment Status 1042 v2.3.1 QA Success 1041 v2.3.0 QA Success
Producción PROD
1
Ejecutar Pipeline de CI/CD
Ingresar al pipeline de entrega continua en Azure DevOps y disparar el release de la rama main con aprobación manual requerida.
Organización : contoso-prod Proyecto : MiApp-PROD Pipeline : deploy-production Rama : main Acción : Run pipeline → requiere aprobación de Release Manager
2
Validar ejecución del pipeline
Esperar aprobación manual en la etapa de Gate y que todas las etapas finalicen con estado Succeeded. Monitorear rollback automático si falla.
⏳ Stage: Build → Succeeded ⏳ Stage: Integration Tests → Succeeded ⏳ Stage: Docker Build → Succeeded ⏳ Stage: Push to ACR → Succeeded ⏸ Stage: Approval Gate → Waiting approval... ✓ Stage: Deploy to PROD → Succeeded
3
Ingresar al Portal de Azure — Recurso PROD
Navegar al portal de Azure y ubicar el grupo de recursos del ambiente de Producción con acceso restringido y MFA requerido.
Suscripción : Azure Production (MFA requerido) Resource Group : rg-miapp-prod-westus2 AKS Cluster : aks-miapp-prod-001 Región : West US 2 URL Portal : portal.azure.com → Kubernetes services
4
Ingresar al AKS — Validar Pods desplegados
Conectarse al clúster AKS de Producción mediante kubectl y validar que los pods del namespace prod estén en estado Running sin reinicios.
# Conectar al cluster az aks get-credentials \ --resource-group rg-miapp-prod-westus2 \ --name aks-miapp-prod-001 # Validar pods en namespace prod kubectl get pods -n prod -o wide # Resultado esperado (3 réplicas por servicio) NAME READY STATUS RESTARTS miapp-api-9f2c4d-ab3kx 1/1 Running 0 miapp-api-9f2c4d-mn7qz 1/1 Running 0 miapp-api-9f2c4d-wx1pv 1/1 Running 0 miapp-worker-5a7e1b-cr4nt 1/1 Running 0 miapp-worker-5a7e1b-jk8ys 1/1 Running 0 miapp-scheduler-8d3f6c-lp2mw 1/1 Running 0
5
Ingresar a la Base de Datos — PROD
Conectarse al servidor de base de datos de Producción (conexión a través de Private Endpoint) y ejecutar una consulta de validación post-deploy.
Servidor : sql-miapp-prod.database.windows.net Base datos: db-miapp-prod Usuario : sqladmin-prod (via Key Vault) Puerto : 1433 (Private Endpoint) -- Query de validación SELECT TOP 10 d.DeploymentId, d.Version, d.Environment, d.DeployedAt, d.Status FROM dbo.Deployments d WHERE d.Environment = 'PROD' AND d.DeployedAt >= GETDATE() - 1 ORDER BY d.DeployedAt DESC; -- Resultado esperado DeploymentId Version Environment Status 5187 v2.3.1 PROD Success 5186 v2.2.9 PROD Success
